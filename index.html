BDD can help

Behavior Driven Development is about building software by describing its behaviors from the perspective of its stakeholders

It describes a cycle of interactions with well defined outputs, resulting in the delivery of working, tested software, that actually matters.

BDD perks

  • Clear communication
  • Creates a shared language
  • Captures intent of requirements
  • More time on actual business needs

Additional benefits

  • Creates clear communication between engineers and business professionals using a shared language
  • Captures intent and desired outcome of requirements in clear English (in our case)
  • Ensures all development effort is spent on delivering on actual business needs
  • design software that actually meets business needs
  • Meets the needs of the USER
  • Improved code quality, reducing time lost on maintenance

It's All About Behaviors

  • Technology is enabler
  • Wait no it's not. Technology is an inhibitor
  • Technology should support human behaviors

Businesses should be focusing on their customers and business strategies, not technology.

Think of an online shop. It doesn't bring revenue in just by existing. It's users do. Software accomplishes business goals by supporting human behaviors.

The magic words


Given [initial context for the example]
When [action that the actor in the system or stakeholder performs]
Then [expected outcome of that action]

Drupal Example with the magic words

I want to be able to create and edit content on my site to inform the world that dogs are objectively better then cats.


Given I using my website
When I go to the login page

User Login

Slap it in a feature file and boom you have your test.


#./features/user_login.feature
@api
Feature: User Login.
  As a an anonymous user,
  I want to be able to log into my website,
  So that I can edit my content.

Scenario: I can log in.
  Given I am an anonymous user
  When I am at "user/login"
  And I fill in "name" with "admin"
  And I fill in "pass" with "admin"
  And press "Log in"
  Then I should see the link "Log out"

Scenario: I can edit content
  Given I am logged in as a user with the administrator role
  When I am at "node/add/page"
  Then I should see "Create Basic page"
  And I should get a 200 HTTP response
  • Top Section: Describe the feature. Provides human context
  • Each feature is described by multiple scenarios - How the feature should act under certain conditions
  • Each of the scenario lines map to function that defines what actions to take.

Executing tests


$ bin/behat features/user_login.feature

/**
 * Asserts that a given module exists and is enabled.
 *
 * @Given the :module module is installed
 */
 public function assertModuleExists($module)
 {
   if (module_exists($module)) {
   return TRUE;
 }

  $message = sprintf('Module "%s" is not installed.', $module);
  throw new \Exception($message);
}

/**
 * Asserts the given migration is complete.
 *
 * @Then /^the "([^"]*)" migration is complete$/
 *
 * @throws /Exception if the migration did not complete.
 */
 public function assertMigrationIsComplete($name)
 {
   $migration = Migration::getInstance($name);
   if ($migration->isComplete()) {
     return TRUE;
   }
   $message = sprintf('The "%s" migration is not complete.', $name);
   throw new \Exception($message);
}
